---
title: Button
description: Button is the standard action trigger for Meteor interfaces. Use it when people need to submit, confirm, continue, or trigger an action directly from the UI.
---

## When to use

- Use **Button** for clear user actions such as saving, creating, confirming, or navigating to the next step.
- Use the `variant` to communicate action hierarchy and intent.
- Use button sizes to fit the surrounding layout without changing the meaning of the action.

## Import

```ts
import { MtButton } from "@shopware-ag/meteor-component-library";
```

## Examples

### Basic

```vue
<template>
  <mt-button variant="primary">Save</mt-button>
</template>
```

### Variants

```vue
<template>
  <mt-button variant="primary">Primary</mt-button>
  <mt-button variant="secondary">Secondary</mt-button>
  <mt-button variant="critical">Delete</mt-button>
  <mt-button variant="ghost">Ghost</mt-button>
</template>
```

### Sizes

```vue
<template>
  <mt-button size="default">Default</mt-button>
  <mt-button size="small">Small</mt-button>
</template>
```

### With icon

```vue
<template>
  <mt-button variant="primary">
    <template #icon-front>
      <mt-icon name="regular-plus-circle-s" />
    </template>
    Add item
  </mt-button>
</template>
```

### Disabled and loading states

```vue
<template>
  <mt-button variant="primary" disabled>Disabled</mt-button>
  <mt-button variant="primary" is-loading>Loading</mt-button>
</template>
```

## Props

| Prop         | Type                                                | Default       | Description                            |
| ------------ | --------------------------------------------------- | ------------- | -------------------------------------- |
| `variant`    | `"primary" \| "secondary" \| "critical" \| "ghost"` | `"secondary"` | Visual style and semantic emphasis     |
| `size`       | `"default" \| "small"`                              | `"default"`   | Button height and padding              |
| `disabled`   | `boolean`                                           | `false`       | Prevents interaction                   |
| `is-loading` | `boolean`                                           | `false`       | Shows a loading indicator              |
| `square`     | `boolean`                                           | `false`       | Renders as an icon-only square control |
| `block`      | `boolean`                                           | `false`       | Expands to fill available inline space |

## Do and don't

**Do**

- Keep button labels short, specific, and action-oriented.
- Use `primary` for the main action in a local context.
- Use `secondary` or `critical` for supporting or destructive actions.

**Don't**

- Do not use multiple primary buttons in the same action group.
- Do not use vague labels like "Submit" when a clearer verb would help.
- Do not use **Button** for purely navigational inline links.
