## What is MCP?

The Model Context Protocol (MCP) is a standardized way for AI assistants to access external data and tools. Meteor ships an MCP server that lets assistants such as Claude Code, Cursor, and VS Code read the documentation directly, so answers about components and guidelines stay grounded in the current docs instead of model memory.

The server uses HTTP transport and is available at:

```text
https://meteor.shopware.com/mcp
```

## Available tools

| Tool | Description |
| ----------- | ------------------------------------------------------------------- |
| `list-pages` | Lists all available documentation pages with their categories and basic information. |
| `get-page`   | Retrieves the full content and details of a specific documentation page. |

## Configuration

Add the server to your AI assistant using the snippets below.

### Claude Code

```bash
claude mcp add --transport http meteor https://meteor.shopware.com/mcp
```

### Cursor

Create or update `.cursor/mcp.json` in your project root:

```json [.cursor/mcp.json]
{
  "mcpServers": {
    "meteor": {
      "type": "http",
      "url": "https://meteor.shopware.com/mcp"
    }
  }
}
```

### VS Code

Create or update `.vscode/mcp.json` in your project:

```json [.vscode/mcp.json]
{
  "servers": {
    "meteor": {
      "type": "http",
      "url": "https://meteor.shopware.com/mcp"
    }
  }
}
```

### Claude Desktop

Open Settings, go to Developer, click Edit Config, and add the server to `claude_desktop_config.json`:

```json [claude_desktop_config.json]
{
  "mcpServers": {
    "meteor": {
      "command": "npx",
      "args": ["mcp-remote", "https://meteor.shopware.com/mcp"]
    }
  }
}
```

Restart Claude Desktop afterwards so the server is registered.

### Windsurf

Add the server to your Windsurf MCP config:

```json [mcp_config.json]
{
  "mcpServers": {
    "meteor": {
      "type": "http",
      "url": "https://meteor.shopware.com/mcp"
    }
  }
}
```

## Usage

Once the server is connected, you can ask your assistant questions like:

- "List the Meteor documentation pages"
- "Show me the mt-button documentation from Meteor"
- "What does the Meteor migration guide say about replacing sw- components?"

The assistant calls `list-pages` to discover what exists, then `get-page` to read the relevant page.

::tip
Prefer structured docs over plain web access: the MCP server returns the same content that powers this site, so the assistant works from the exact, current documentation.
::
