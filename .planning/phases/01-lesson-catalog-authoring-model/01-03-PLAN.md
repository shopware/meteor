<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifest file references -> raw asset loader | Lesson metadata controls which local files are exposed to the tutorial runtime |
| normalized catalog -> future UI consumers | Browser UI will trust the loader output for ordering, docs links, and file-role semantics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `packages/admin-sdk-tutorial/src/catalog/loadLessonCatalog.ts` | mitigate | Reject manifest references containing `..` or leading `/`, and only load assets matched by the explicit `../lessons/**/*.{md,ts,js,vue,json}` raw glob |
| T-03-02 | Information Disclosure | `packages/admin-sdk-tutorial/src/catalog/loadLessonCatalog.ts` | mitigate | Limit exposed lesson assets to `proseFile`, `starterFile`, `solutionFile`, and `supportFiles` declared by the manifest so unrelated repo files cannot be pulled into the runtime bundle |
| T-03-03 | Denial of Service | `packages/admin-sdk-tutorial/src/catalog/normalizeCatalog.ts` | mitigate | Throw on duplicate ids or conflicting order metadata and cover those cases with unit tests so the browser cannot render nondeterministic lesson sequences |
</threat_model>
