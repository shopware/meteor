<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Repo-authored lesson files -> manifest contract | Lesson metadata can introduce invalid file roles or unsafe docs links if authoring rules are loose |
| Canonical docs metadata -> future UI | Link targets leave the app and must stay on trusted docs destinations |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `lesson.manifest.ts` files | mitigate | Keep manifest file references lesson-local only and cover `primaryEditableFile`, `starterFile`, `solutionFile`, and `supportFiles` with real spec assertions |
| T-02-02 | Spoofing | `docsLinks` metadata | mitigate | Restrict docs links to canonical `/guide/...` routes and assert that pattern in `catalog.spec.ts` |
| T-02-03 | Information Disclosure | Lesson prose and support files | accept | The lesson assets are intentionally repo-authored teaching content with no secret data |
</threat_model>
