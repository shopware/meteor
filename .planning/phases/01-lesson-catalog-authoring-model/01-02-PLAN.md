<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| manifest file references -> raw asset loader | Lesson metadata controls which local files are exposed to the tutorial runtime |
| normalized catalog -> future UI consumers | Browser UI will trust the loader output for ordering, docs links, and file-role semantics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | T | `packages/admin-sdk-tutorial/src/catalog/loadLessonCatalog.ts` | mitigate | Reject manifest references containing `..` or leading `/`, and only load assets matched by the explicit `../lessons/**/*.{md,ts,js,vue,json}` raw glob |
| T-01-05 | I | `packages/admin-sdk-tutorial/src/catalog/loadLessonCatalog.ts` | mitigate | Limit exposed lesson assets to `proseFile`, `starterFile`, `solutionFile`, and `supportFiles` declared by the manifest so unrelated repo files cannot be pulled into the runtime bundle |
| T-01-06 | D | `packages/admin-sdk-tutorial/src/catalog/normalizeCatalog.ts` | mitigate | Throw on duplicate ids or conflicting order metadata and cover those cases with unit tests so the browser cannot render nondeterministic lesson sequences |
</threat_model>
