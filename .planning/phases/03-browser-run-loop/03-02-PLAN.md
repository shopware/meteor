<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sandboxed iframe -> host window | Learner code crosses into the tutorial shell only through `postMessage` |
| active preview session -> restarted preview session | Old preview windows and tokens must become invalid immediately after restart |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Spoofing | `packages/admin-sdk-tutorial/src/runtime/preview-bridge.ts` | mitigate | Require all preview traffic to match the active iframe window, expected origin, `sessionId`, and `sessionToken` before the host trusts it |
| T-03-05 | Elevation of Privilege | `packages/admin-sdk-tutorial/src/runtime/preview-session.ts` | mitigate | Create preview iframes with `sandbox="allow-scripts"` only and explicitly forbid `allow-same-origin`, popups, forms, or top-navigation escapes |
| T-03-06 | Denial of Service | `packages/admin-sdk-tutorial/src/runtime/preview-session.ts` | mitigate | Replace the entire iframe on restart and destroy old listeners so timers, callbacks, or stale windows cannot keep mutating runtime state |
| T-03-07 | Tampering | `packages/admin-sdk-tutorial/src/runtime/preview-host.ts` | mitigate | Accept only the exact SDK message `_type`s Phase 3 needs and return an error for unknown requests instead of silently widening the bridge |
</threat_model>
