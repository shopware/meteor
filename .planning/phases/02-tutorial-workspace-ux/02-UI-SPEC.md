## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, dirty-state dots, compact metadata spacing |
| sm | 8px | Button icon gaps, list item padding steps, compact toolbar spacing |
| md | 16px | Default control spacing, body copy rhythm, card internals |
| lg | 24px | Guidance card padding, editor toolbar padding, panel gaps |
| xl | 32px | Desktop shell gutter, major pane padding |
| 2xl | 48px | Large content breaks inside lesson prose |
| 3xl | 64px | Page-level outer spacing on large screens |

Exceptions: minimum 44px interactive height for lesson rows, mobile section tabs, and icon-only controls; preview placeholder minimum height 160px on narrow screens and 192px on desktop

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Open lesson` |
| Empty state heading | `No lessons available yet` |
| Empty state body | `Add or restore a lesson bundle to load guidance, starter code, and the preview placeholder.` |
| Error state | `This lesson could not be loaded. Reload the tutorial. If the problem continues, return to the lesson list and open another lesson.` |
| Destructive confirmation | `Restore starter`: `Restore the starter code for this lesson? Your current draft on this device will be replaced.` |

## Layout Contract

- `>= 1024px`: render a two-pane shell. The left guidance pane is 360px wide by default and may grow to 400px on large screens. The right workspace pane fills the remaining width.
- The left pane contains the lesson hierarchy, lesson title, summary, prose, docs links, support-file summary, save-state text, and workspace actions.
- The right pane keeps the editor as the primary surface and the preview placeholder directly below it in the same pane. Do not add a runnable console, fake-admin shell, or execution controls in this phase.
- `< 1024px`: switch to a single-column shell with a sticky top tab set: `Lessons`, `Guide`, `Workspace`. The `Workspace` tab stacks the editor above the preview placeholder.
- Keep pane sizing fixed in Phase 2. Do not introduce draggable splitters yet.
- On first visit, open the first lesson in authored catalog order automatically. On refresh, reopen the last active lesson instead.

## Component Inventory

- `Local workspace shell`: app header, hierarchical lesson rail, editor toolbar, save-status text, support-file summary, preview placeholder, dirty-state badge, and solution compare container.
- `CodeMirror 6`: single editable file only. Show the active file path as a non-interactive label above the editor.
- `MtButton`: `Open lesson`, `Show solution`, `Hide solution`, `Restore starter`, and docs-link secondary actions.
- `MtCard`: guidance blocks, preview placeholder container, and read-only solution surface.
- `MtTabs`: narrow-screen shell tabs and narrow-screen solution switcher.
- `MtEmptyState`: catalog-empty state and preview placeholder body.
- `MtIcon`: docs-link affordances, dirty-state marker support, and status adornments.

Support files stay read-only in this phase. Surface them as a list in the guidance pane; do not render a second editable file tabset.

---

## Interaction Contract

- Persist two things per same-device session: the last active lesson id and the current draft for each visited lesson. Key both by stable lesson identity from the normalized catalog.
- Auto-save draft changes after 750ms idle, on editor blur, and immediately before lesson switches. Show `Saving...` only while a write is pending; otherwise show `Saved on this device`.
- If the current draft differs from the starter state, show a subtle dirty marker next to the lesson title and lesson-row label.
- Switching to another lesson while the current lesson is dirty must open a non-destructive warning dialog. Use copy that makes the local-save behavior clear: `Your draft stays saved on this device. Open the next lesson or keep editing here?`
- `Show solution` opens a secondary read-only comparison surface without mutating learner code. On desktop, split the editor region into a 50/50 compare view. On narrow screens, switch the workspace area to two sub-tabs: `Your code` and `Solution`.
- `Hide solution` returns the learner to the exact working draft. Do not persist solution visibility across refresh.
- `Restore starter` is always destructive. Use a confirmation modal with secondary action `Keep draft` and destructive action `Restore starter`.
- The preview area is a framed placeholder in Phase 2. Title it `Preview` and use neutral helper copy: `Interactive output arrives in Phase 3. For now, focus on reading, editing, and comparing code.`
- Docs links open in a new tab and keep the learner in the tutorial workspace.

---

## Accessibility Contract

- Lesson navigation, workspace tabs, and action buttons must be fully keyboard reachable.
- Mark the active lesson row with `aria-current="page"`.
- Use semantic button elements for lesson switching and actions; do not rely on clickable `div` containers.
- Any icon-only control must provide an `aria-label` and either a tooltip or adjacent text label fallback so its purpose stays visible and screen-reader safe.
- Preserve visible focus rings in the accent color for the active interactive element.
- Confirmation dialogs must trap focus and return focus to the triggering control when closed.
