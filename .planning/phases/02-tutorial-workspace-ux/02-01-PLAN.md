<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser storage -> workspace restore | Persisted lesson id and draft JSON can be stale, malformed, or user-edited |
| normalized catalog -> active lesson lookup | Stored lesson ids must map only to real lessons returned by the current catalog |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `packages/admin-sdk-tutorial/src/workspace/storage.ts` | mitigate | Parse storage defensively, require object-shaped draft maps, and fall back to empty state on malformed JSON or invalid payload types |
| T-01-02 | Denial of Service | `packages/admin-sdk-tutorial/src/workspace/useTutorialWorkspace.ts` | mitigate | Ignore missing or stale lesson ids and reopen the first authored lesson instead of throwing during app bootstrap |
| T-01-03 | Tampering | `packages/admin-sdk-tutorial/src/workspace/useTutorialWorkspace.ts` | mitigate | Persist only lesson ids and raw drafts; never persist compare-mode or destructive-action state that could reopen the UI in a surprising state |
</threat_model>
