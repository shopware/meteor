| lesson markdown -> rendered guidance UI | Trusted repo-authored lesson prose becomes visible structured guidance |
| learner draft -> editor surface | User-entered code must stay isolated to the lesson editor and not imply runnable behavior |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | `packages/admin-sdk-tutorial/src/workspace/lessonProse.ts` | mitigate | Only render repo-authored markdown structure and do not pass arbitrary HTML through to the DOM |
| T-02-02 | Spoofing | `packages/admin-sdk-tutorial/src/App.vue` | mitigate | Keep the preview area explicitly labeled as a placeholder and exclude run controls so users do not mistake Phase 2 for an executing runtime |
| T-02-03 | Denial of Service | `packages/admin-sdk-tutorial/src/App.vue` | mitigate | Reuse the single `useTutorialWorkspace()` source of truth instead of reloading the full catalog on every tab or editor interaction |
</threat_model>
