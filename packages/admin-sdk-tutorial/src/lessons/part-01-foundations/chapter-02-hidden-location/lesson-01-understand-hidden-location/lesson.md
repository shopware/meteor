# Understand the hidden location

This lesson makes the hidden iframe tangible before the tutorial starts rendering anything visual.

The starter code asks you to distinguish two jobs. In the hidden location you register extension points and choose the `positionId` that should host future UI. In the visible location you render the content for the `locationId` you created earlier.

Work through the branches until the intent is obvious from the code alone. You should be able to answer two questions after the exercise: where does bootstrap logic belong, and where does the rendered lesson surface belong?

Use the linked concept pages once the split feels clear. They explain the broader rules around locations and the position IDs you target from the hidden runtime.
