# Dispatch your first notification

In this lesson you wire the smallest useful feedback loop in the Admin SDK: a notification that tells the user something just happened.

Start with the provided starter file and make the payload feel intentional. The goal is not to exhaust every option. It is to notice which fields change the tone of the message and which ones change where it appears.

When you run the lesson, aim for a message that would make sense after a successful action in an extension. Keep the title short, make the body specific, and choose a variant that matches the outcome you want to teach.

After you are comfortable with the payload, compare your result with the solution and then use the linked reference to explore the optional notification fields in more depth.
